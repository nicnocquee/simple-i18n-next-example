# Über

Hallo! Dies ist eine Beispiel-Markdown-Datei, um zu demonstrieren, wie man Markdown-Dateien mit simple-i18n-next verwendet. Warum nur JSON-Dateien verwenden, wenn man auch verschiedene Markdown-Dateien für verschiedene Sprachen bereitstellen kann?

Dieses Markdown kann eine `userId`-Eigenschaft erhalten, die derzeit <code>{props.userId}</code> ist. Die Seitenkomponente, die dieses Markdown rendert, erhält die `userId` aus dem Pfadnamen und übergibt sie an die Markdown-Komponente. Schau dir diese Seite an, die die `userId` `42` hat: <a href={props.demoLink}>{props.demoLink}</a>.

**Es ist auch vollkommen typensicher.** Das CLI generiert die erforderlichen Eigenschaften für die Markdown-Komponente, damit du sie sicher in deinen Seiten verwenden kannst. Zum Beispiel, wenn du keine `userId`-Eigenschaft an die Markdown-Komponente übergibst, wird ein Fehler ausgegeben.
